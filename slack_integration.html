Notification rules — what fires, where, and when
Automated Notification Rules
Trigger Channel / Target Type Content
Every scored demo #demo-reviews Auto Full notification with score, verdict, SPICED status, deal outcome, and scorecard link
Score ≥ 80 #killer-calls 80+ only Celebratory notification with rank, framework highlights, and "Study This Call" link
Score < 60 #demo-reviews Red flag Standard notification with red attachment border. No special shaming — same format, different color.
2+ consecutive reds Manager DM Alert Private coaching alert with pattern analysis across recent calls, common failure points, and recommended action
New #1 All Time score #killer-calls Conditional Special "New #1" announcement with comparison to previous record holder
Rep breaks personal best #demo-reviews Auto Standard notification with "Personal Best" tag appended — visible to the team as a milestone
Deal outcome updates (Pipedrive) #demo-reviews thread Auto Reply in the original demo's thread: "Update: This deal closed won — $8,400 ARR" or "Update: Deal marked lost"
Discount given before ECIR #demo-reviews Red flag Auto-flagged in scorecard regardless of overall score. Notification includes the specific timestamp where discount was offered.
🔑
Authentication: Slack Identity Only
No separate accounts, passwords, or logins. Every comment, reaction, and interaction is attributed via the team's existing Slack identities. The dashboard is read-only for engagement data — all posting and discussion happens in Slack. This means zero onboarding friction and the team stays in the tool they already live in.
